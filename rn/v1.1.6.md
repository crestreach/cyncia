# cyncia 1.1.6

Patch release on top of [v1.1.5](v1.1.5.md).
Adds the Cyncia logo and a guideline against reconstructing `.agent-config/` assets from generated outputs.

## Highlights

- **Cyncia logo.**
  Added the project logo (wordmark + slogan) under [`assets/logo/`](../assets/logo/), embedded as SVG in [`README.md`](../README.md).
  The default set uses Satoshi Bold; alternate typeface treatments (General Sans, Figtree) are kept in [`assets/logo/generalsans/`](../assets/logo/generalsans/) and [`assets/logo/figtree/`](../assets/logo/figtree/) for comparison.
- **New guideline: don't simulate `.agent-config/` assets from outputs.**
  The existing guidance only forbade hand-editing generated files; it did not stop an assistant from reconstructing source assets out of generated outputs instead of syncing.
  [`.agent-config/AGENTS.md`](../.agent-config/AGENTS.md) (fanned out to root `AGENTS.md`, `CLAUDE.md`, `AGENTS.override.md`, and `.github/copilot-instructions.md`) now says explicitly: unless asked otherwise, sync `.agent-config/` changes first rather than simulating them from generated outputs.

## Fixes / cleanups

- Capitalize the `Cyncia` project heading in [`README.md`](../README.md).

## Tests

No script behavior changed in this release; the existing bats/Pester suites remain green.
