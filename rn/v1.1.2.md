# cyncia 1.1.2

Patch release on top of [v1.1.1](v1.1.1.md). Removes the explicit
`model: inherit` setting from the bundled and example `aside` agent source,
letting tools that inherit by omission use their native default model behavior.

## Highlights

- **Copilot-compatible aside agent.** The authored `aside` agent no longer
  emits an explicit `model: inherit` value in the generic source or example
  source, avoiding unsupported model metadata in generated Copilot agents.
- **Native default model behavior.** Tools that inherit the current or default
  model by omission can now do so without carrying a tool-specific sentinel
  value in the portable source files.

## Fixes / cleanups

- Remove `model: inherit` from `.agent-config/agents/aside.md`.
- Remove `model: inherit` from `examples/agents/aside.md`.

## Tests

Validated before release:

- `git --no-pager diff --check`
- Verified no `aside` agent files still contain `model: inherit`.

## Upgrading from 1.1.1

Re-run the installer to refresh `.cyncia/`, then re-run `sync-all` in projects
that use the bundled `aside` agent source:

```bash
curl -fsSL https://raw.githubusercontent.com/crestreach/cyncia/main/install/install.sh | bash
```
