# cyncia 1.1.3

Patch release on top of [v1.1.2](v1.1.2.md).
The installer now ships the project `LICENSE` into the installed `.cyncia/` directory, and the `markdown-style` rule adopts semantic line breaks instead of fixed-column wrapping.
All authored Markdown docs are reformatted to match.

## Highlights

- **License travels with the install.**
  `install.sh` now validates and copies `LICENSE` from the downloaded snapshot into `<cyncia-dir>/` alongside `README.md` and `cyncia.md`, so the installed cyncia files always carry their license.
- **Semantic line breaks in `markdown-style`.**
  The rule now recommends starting a new line at sentence/clause boundaries instead of hard-wrapping prose to a fixed column, keeping diffs small and avoiding reflow churn when editing.
- **Docs reformatted to semantic line breaks.**
  All authored Markdown docs and config (`README.md`, `cyncia.md`, `tools.md`, `.agent-config/`, `examples/`, and more) were reflowed to one sentence per line, applying the new rule across the repository.

## Fixes / cleanups

- Add `LICENSE` to the installer's required-entry validation, copy step, header comment, and the "wrote/refreshed" console summary.
- Update the `cyncia.md` installer description to list `LICENSE` among the copied files.
- Replace the fixed ~100-column wrap guidance in `.agent-config/rules/markdown-style.md` with semantic line breaks.
- Reflow all authored Markdown docs and config to semantic line breaks (whitespace/newline-only changes) and regenerate the per-tool outputs via `sync-all`.

## Tests

Validated before release:

- `test/run-bats.sh` — full bats suite (87 tests) passing.
- Install tarball builders updated to include `LICENSE`; added a positive assertion that `LICENSE` lands in `.cyncia/` and a negative test that a snapshot missing `LICENSE` errors.
