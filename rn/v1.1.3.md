# cyncia 1.1.3

Patch release on top of [v1.1.2](v1.1.2.md). The installer now ships the
project `LICENSE` into the installed `.cyncia/` directory, and the
`markdown-style` rule adopts semantic line breaks instead of fixed-column
wrapping.

## Highlights

- **License travels with the install.** `install.sh` now validates and copies
  `LICENSE` from the downloaded snapshot into `<cyncia-dir>/` alongside
  `README.md` and `cyncia.md`, so the installed cyncia files always carry their
  license.
- **Semantic line breaks in `markdown-style`.** The rule now recommends starting
  a new line at sentence/clause boundaries instead of hard-wrapping prose to a
  fixed column, keeping diffs small and avoiding reflow churn when editing.

## Fixes / cleanups

- Add `LICENSE` to the installer's required-entry validation, copy step, header
  comment, and the "wrote/refreshed" console summary.
- Update the `cyncia.md` installer description to list `LICENSE` among the
  copied files.
- Replace the fixed ~100-column wrap guidance in
  `.agent-config/rules/markdown-style.md` with semantic line breaks.

## Tests

Validated before release:

- `test/run-bats.sh` — full bats suite (87 tests) passing.
- Install tarball builders updated to include `LICENSE`; added a positive
  assertion that `LICENSE` lands in `.cyncia/` and a negative test that a
  snapshot missing `LICENSE` errors.

## Upgrading from 1.1.2

Re-run the installer to refresh `.cyncia/` (it now also installs `LICENSE`):

```bash
curl -fsSL https://raw.githubusercontent.com/crestreach/cyncia/main/install/install.sh | bash
```
