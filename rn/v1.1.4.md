# cyncia 1.1.4

Patch release on top of [v1.1.3](v1.1.3.md).
Fixes a mawk regex panic in `heading_level()` that could truncate merged `CLAUDE.md` and `.junie/AGENTS.md` output.

## Highlights

- **Reliable rule merging under mawk.**
  When generating Claude Code and Junie guidelines, `sync-agent-guidelines` shifts the headings of rule bodies merged into `CLAUDE.md` / `.junie/AGENTS.md`.
  Under `mawk` (the default `awk` on Debian/Ubuntu), the bounded interval regex `/^#{1,6}([ \t]|$)/` in `heading_level()` triggered a `REcompile` panic, which aborted the awk pass and truncated the merged output.
  The level detector now uses an interval-free pattern with an explicit guard, so heading normalization compiles and runs correctly across `awk` implementations.

## Fixes / cleanups

- Replace the bounded interval `/^#{1,6}([ \t]|$)/` in `scripts/common/common.sh` with the interval-free `/^#+([ \t]|$)/` plus an explicit `n > 6` guard, so `heading_level()` compiles under `mawk` instead of panicking.

## Tests
