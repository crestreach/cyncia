# cyncia 1.0.0

First stable release of cyncia — the tool-agnostic source-of-truth for AI
coding-assistant configuration.

Website: <https://www.cyncia.net>

## Highlights

- **One source tree, five tools.** Author your AI assistant config once under
  `.agent-config/` and generate the per-tool layouts that Cursor, Claude Code,
  GitHub Copilot, VS Code, and JetBrains Junie expect — file paths, formats,
  and frontmatter keys all translated for you.
- **Curl-pipeable installer.**
  `curl -fsSL https://raw.githubusercontent.com/crestreach/cyncia/main/install/install.sh | bash`
  bootstraps `.agent-config/`, fetches a snapshot of cyncia into `.cyncia/`,
  and (by default) copies the bundled skills + runs `sync-all`.
- **Optional source folders.** Only `AGENTS.md` is required; missing
  `agents/`, `skills/`, `rules/`, or `mcp-servers/` are skipped with a
  console note.
- **MCP server sync** with per-tool secret-token translation:
  `${secret:NAME}` → Cursor `${env:NAME}`, Claude `${NAME}`, VS Code
  `${input:NAME}` (with `inputs[]`), and a stdout-only snippet for Junie.
- **Per-agent MCP allowlists** translated across Cursor, Claude, Copilot,
  and Junie.
- **Tested.** 65 bats tests + 31 Pester tests covering `sync-all`, per-tool
  sync scripts, MCP translation, optional folders, and the installer
  end-to-end.

## What's in this release

### Sync engine

- `scripts/sync-all.{sh,ps1}` — one entry point that fans guidelines, agents,
  skills, rules, and MCP servers out to all five supported tools.
- Per-tool scripts under `scripts/{cursor,claude,copilot,vscode,junie}/` for
  `sync-agent-guidelines`, `sync-agents`, `sync-skills`, `sync-rules`, and
  `sync-mcp`.
- `--items` for partial syncs, `--clean` for stale-output removal, `--tools`
  for tool scoping.

### Generated layouts

| Source                 | Cursor                   | Claude Code               | GitHub Copilot                             | VS Code            | JetBrains Junie                    |
| ---------------------- | ------------------------ | ------------------------- | ------------------------------------------ | ------------------ | ---------------------------------- |
| `AGENTS.md`            | `AGENTS.md`              | `CLAUDE.md`               | `.github/copilot-instructions.md`          | —                  | `.junie/AGENTS.md`                 |
| `agents/<n>.md`        | `.cursor/agents/<n>.md`  | `.claude/agents/<n>.md`   | `.github/agents/<n>.md`                    | —                  | `.junie/agents/<n>.md`             |
| `skills/<n>/…`         | `.cursor/skills/<n>/…`   | `.claude/skills/<n>/…`    | `.github/skills/<n>/…`                     | —                  | `.junie/skills/<n>/…`              |
| `rules/<n>.md`         | `.cursor/rules/<n>.mdc`  | (merged into `CLAUDE.md`) | `.github/instructions/<n>.instructions.md` | —                  | (merged into `.junie/AGENTS.md`)   |
| `mcp-servers/<n>.json` | `.cursor/mcp.json`       | `.mcp.json`               | —                                          | `.vscode/mcp.json` | (stdout snippet)                   |

### Installer (`install/install.sh`)

- Creates the `.agent-config/` skeleton (subdirs + stub `AGENTS.md`) without
  overwriting an existing `AGENTS.md`.
- Downloads `https://github.com/<repo>/archive/<ref>.tar.gz` into
  `<cyncia-dir>/` (default `.cyncia/`); re-running refreshes `scripts/`,
  `skills/`, `examples/` so upstream deletions propagate.
- Writes `<cyncia-dir>/VERSION` with the installed ref. For the default
  `main` branch the installer best-effort queries the GitHub API for tags
  pointing at HEAD and lists those instead.
- Two interactive prompts (copy bundled skills; run `sync-all`) default to
  **yes** — pressing Enter accepts; pass `--no-bootstrap` to decline both.
  Without a TTY (`curl | bash` with no controlling terminal) the script
  also defaults to yes.
- Flags: `--config-dir`, `--cyncia-dir`, `--ref`, `--repo`, `--bootstrap`,
  `--no-bootstrap`. Env: `CYNCIA_REPO`, `CYNCIA_REF`.

### Tests

- bats-core suite under `test/bats/` (65 tests): `sync-all`,
  items/clean/tools, agent-frontmatter MCP translation, MCP per-tool token
  translation, optional folders, installer end-to-end (with an offline
  `curl` shim).
- Pester v5 suite under `test/pester/` (31 tests) for the PowerShell sync
  path.

## Installation

```bash
curl -fsSL https://raw.githubusercontent.com/crestreach/cyncia/main/install/install.sh | bash
```

To pin this release:

```bash
curl -fsSL https://raw.githubusercontent.com/crestreach/cyncia/main/install/install.sh \
  | bash -s -- --ref v1.0.0
```

See [`README.md`](../README.md) for a quickstart and [`cyncia.md`](../cyncia.md)
for the full reference (formats, frontmatter, MCP secret tokens, per-tool
field map, edge cases).

## Required tools

`bash` 4+, `curl`, `tar`, `mktemp`, `find`, `awk` for the installer;
`jq` 1.6+ only for the MCP sync step. PowerShell 5.1+/7+ for the `*.ps1`
scripts. Git is **not** required by the installer or at sync time.
