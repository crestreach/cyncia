# cyncia 1.1.1

Patch release on top of [v1.1.0](v1.1.0.md). Aligns GitHub Copilot
workspace custom-agent filenames with the native `.agent.md` convention,
keeps the generic source format unchanged, and refreshes docs/tests around
the generated output matrix.

## Highlights

- **Copilot custom-agent filenames.** Copilot sync now emits workspace agents
  as `.github/agents/<name>.agent.md`, matching the VS Code / GitHub Copilot
  custom-agent convention.
- **Clean refresh recommended.** Existing installs should refresh with a clean
  sync so stale `.github/agents/<name>.md` files do not remain beside the new
  `.github/agents/<name>.agent.md` outputs as duplicate Copilot agents.
- **Portable source stays stable.** Generic source agents remain authored as
  `agents/<name>.md`; only the Copilot generated output path changes.
- **Per-tool docs clarified.** README, `cyncia.md`, `tools.md`, and generated
  agent guidance now document the current per-tool agent filename conventions:
  Cursor, Claude Code, and Junie use Markdown files; Copilot uses `.agent.md`;
  Codex uses TOML.

## Fixes / cleanups

- Update Bash and PowerShell Copilot agent sync scripts to write
  `.github/agents/<name>.agent.md`.
- Update VS Code agent-sync no-op messaging to point readers at Copilot's
  `.github/agents/*.agent.md` workspace-agent output.
- Add a reusable release prompt under `rn/release-prompt.md`.

## Tests

- Bats coverage now asserts Copilot `.agent.md` outputs and the absence of the
  old `.github/agents/<name>.md` path.
- Pester coverage now checks the same Copilot agent filename behavior for the
  PowerShell sync path.

## Upgrading from 1.1.0

Re-run the installer to refresh `.cyncia/`, then re-run `sync-all --clean` so
Copilot agent outputs are regenerated with the `.agent.md` filename and stale
`.github/agents/<name>.md` files are removed:

```bash
curl -fsSL https://raw.githubusercontent.com/crestreach/cyncia/main/install/install.sh | bash
```
