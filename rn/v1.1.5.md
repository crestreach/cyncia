# cyncia 1.1.5

Patch release on top of [v1.1.4](v1.1.4.md).
Adds fail-fast validation of requested tools in `sync-all`, normalizes error message prefixes, and tidies docs.

## Highlights

- **Fail-fast tool validation in `sync-all`.**
  `sync-all.sh` / `sync-all.ps1` now validate every name in `--tools` / `-Tools` *before* running any sync step.
  Previously a typo such as `--tools cursor,cusror` synced `cursor` in full and only then aborted on the unknown name, leaving a half-written output root.
  Now an unknown tool errors immediately and nothing is generated.

## Fixes / cleanups

- Validate all requested tools up front in [`scripts/sync-all.sh`](../scripts/sync-all.sh) and [`scripts/sync-all.ps1`](../scripts/sync-all.ps1) so a typo fails before any output is written.
- Normalize error message prefixes to `Error:` across [`install/install.sh`](../install/install.sh) and the Copilot agent sync scripts for consistent output.
- Add the pronunciation of *Cyncia* to [`README.md`](../README.md) and [`cyncia.md`](../cyncia.md).
- Use Cyncia-relevant paths in the comment examples of [`rn/generate-release-notes.sh`](../rn/generate-release-notes.sh).

## Tests

Validated before release:

- `test/run-bats.sh` — full bats suite (89 tests) passing.
- `test/run-pester.ps1` — full Pester suite (44 tests) passing.
